# 源码文件说明

> ⚠️ 本目录存放所有项目源码，便于离线查看。部署时直接使用 GitHub 仓库即可。

## 目录结构

```
server-src/              # 后端源码 (Express + TypeScript)
├── index.ts             # 入口文件：Express 服务器、所有 API 路由
├── config.ts            # 环境变量配置（从 process.env 读取）
├── routes/
│   ├── wechat.ts        # 微信回调：消息接收、暗号加入、图片处理
│   └── photos.ts        # 照片 REST API：列表、上传、删除、评论
├── services/
│   ├── wechatService.ts # 微信：签名验证、XML解析、access_token、菜单
│   ├── photoService.ts  # 图片处理 Pipeline（压缩 + 上传 Cloudinary）
│   ├── cloudinaryService.ts  # Cloudinary SDK 封装（上传/删除/备份/恢复/用量）
│   ├── authService.ts   # 管理员认证（token 生成/验证）
│   └── r2Service.ts     # Cloudflare R2 SDK（备用方案，当前未使用）
└── models/
    ├── photo.ts         # 照片元数据：CRUD、分页、评论、统计、时区
    ├── member.ts        # 成员白名单：添加/删除/查询
    └── adminConfig.ts   # 管理员密码持久化（V1.1 动态改密）

client-src/              # 前端源码 (React + TailwindCSS + Vite)
├── App.tsx              # 主布局：导航栏、路由
├── main.tsx             # React 入口
├── index.css            # 全局样式：瀑布流、动画、滚动条
├── pages/
│   ├── HomePage.tsx     # 首页：照片墙 + 月份筛选 + 管理工具
│   ├── PortfolioPage.tsx # 作品集：悠悠/之之/大家/探索 四个分类
│   ├── RandomGallery.tsx # 随便看看：随机打乱照片
│   └── HelpPage.tsx     # 操作手册页面
├── components/
│   ├── PhotoWall.tsx    # 瀑布流网格 + 留言数 + 管理员编辑
│   ├── Lightbox.tsx     # 灯箱：大图 + 留言 + 保存到手机
│   ├── UploadButton.tsx # 网页上传按钮
│   ├── AdminPanel.tsx   # 管理员登录弹窗
│   ├── MemberManager.tsx # 成员管理面板
│   ├── TodayBanner.tsx  # 今日动态横幅
│   └── StorageBadge.tsx # 存储容量指示器
└── hooks/
    ├── usePhotos.ts     # 照片数据加载 Hook（分页+分类）
    └── useAdmin.ts      # 管理员状态 Hook（登录/token/鉴权）

新增组件（V1.1）
├── ChangePassword.tsx   # 管理员修改密码面板（V1.1新增）
└── (所有现有组件均已升级至V1.1)

配置文件
├── Dockerfile           # Render 部署用 Docker 镜像定义
├── render.yaml          # Render 一键部署配置
├── server-package.json  # 后端 npm 依赖
├── server-tsconfig.json # TypeScript 编译配置
├── server-env.example   # 环境变量模板
├── client-package.json  # 前端 npm 依赖
├── client-tsconfig.json # TypeScript 编译配置
├── client-index.html    # HTML 入口
├── client-vite.config.ts# Vite 构建配置
├── client-tailwind.config.js  # TailwindCSS 主题
├── client-postcss.config.js   # PostCSS 配置
├── dockerignore         # Docker 忽略文件
└── gitignore            # Git 忽略文件
```

## 核心数据流

```
微信发图 → routes/wechat.ts → services/wechatService.ts (下载)
  → services/photoService.ts (压缩) → services/cloudinaryService.ts (上传)
  → models/photo.ts (入库) → reply 确认语句
```

## 备份恢复链路

```
写入操作 → models/photo.ts save() → 本地 JSON 落盘
  → services/cloudinaryService.ts backupJson() → Cloudinary raw backup

启动时 → 从 Cloudinary 恢复 → 与本地对比 → 取数据量多者
```
