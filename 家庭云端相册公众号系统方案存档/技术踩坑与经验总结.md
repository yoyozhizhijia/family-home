# 技术踩坑与经验总结

> 本文档记录了本项目开发过程中遇到的所有 bug、根因、解决方案。
> 每一条都可以作为后续部署的检查清单。

---

## 🔴 P0 级（数据丢失 / 功能不可用）

### 1. 上传后确认语句消失

**现象**：微信发照片能存上，但收不到任何回复

**根因**：
- 图片处理链路太长：下载(1s) → 压缩(1s) → Cloudinary上传(2s) → 延迟备份(≤6s) → 回复；微信等待超时 5 秒后直接丢弃回复
- 使用 `res.send()` 时 Express 在异步回调中的响应可能未立即 flush 到 socket

**解决方案**：
1. **先回复，后处理**：图片消息到达后立即调用 `xmlReply()` 回复确认语句，其余处理（下载→压缩→入库）异步执行
2. **用 `express.text()` 中间件替代 `req.on('data')`**：避免 Express 内部的异步竞争
3. **xmlReply 统一函数**：`res.setHeader('Content-Type', 'text/xml'); res.status(200).send(xmlStr)`

**注意**：此修复在所有新的部署中都应作为起点——不要走 `req.on('data/end')` 的老路

---

### 2. 同一张照片重复上传 2-3 次

**根因**：微信重试机制。处理图片超过5秒未收到 HTTP 响应，微信自动重发同一消息

**解决方案**：
- 维护一个 `processedMsgIds: Set<string>`，5分钟 TTL
- 收到消息后先检查 `msgId`，已在集合中 → 直接返回 `success`，不处理

---

### 3. Render 重启后所有数据丢失

**根因**：数据仅存在 Render 容器本地磁盘，每次部署/重启/休眠都会清空容器文件

**解决方案**：
1. 照片数据直接存入 Cloudinary（图片永久保存）
2. 元数据（photos.json + members.json）每次 `save` 时同步备份到 Cloudinary raw endpoint
3. 启动时从 Cloudinary 恢复，本地 vs 云端**取数据量多者**
4. **注意**：备份函数用 base64 直接上传 (`data:application/json;base64,xxx`)，不要用 stream 方式（Render 容器内不稳定）

---

### 4. 备份数据比本地旧，启动后丢失了新增的照片

**根因**：`initFromCloud()` 无条件用云端数据覆盖本地，云端是旧的2张，覆盖了本地4张

**解决方案**：
- 比较 `remote.length` vs `local.length`，取多的那个
- 如果本地更多，立刻补备份 `save()` 同步到云端

---

### 5. Cloudinary raw URL 构造错误导致恢复返回404

**根因**：`cloudinary.url()` 对 raw 文件的 URL 格式不正确

**正确格式**：
```
https://res.cloudinary.com/{CLOUD_NAME}/raw/upload/{PUBLIC_ID}.json
```
**示例**：
```
https://res.cloudinary.com/diecwpjww/raw/upload/family-home-data-backup-photos.json
```

## 🟡 P1 级（功能受限）

### 6. 个人订阅号不能调用菜单API (errcode: 48001)

**根因**：微信个人订阅号没有 API 创建菜单的权限，只能用后台手工设置

**结论**：不要试图通过 API 设置菜单，关注自动回复替代即可

---

### 7. 个人订阅号不支持 `<a>` 标签

**根因**：微信被动回复消息中 `<a href>` 标签在个人号上不稳定，可能被静默丢弃

**解决方案**：全部改为裸 URL，微信会自动识别为可点击链接

---

### 8. 注册人不能给自己的公众号发消息

**根因**：微信机制——公众号管理员点进去看到的是后台，不是聊天窗口

**解决方案**：用任意一个家人的微信注册公众号，确保其他人（包括管理员本人）都能正常发图

---

### 9. Express 全局 `express.json()` 可能干扰 XML body 读取

**根因**：全局的 body-parser 中间件可能在处理 XML 时改变 content-type

**解决方案**：微信路由上显式使用 `express.text({ type: 'text/xml' })` 作为前置中间件

---

## 🟢 P2 级（体验优化）

### 10. 手机端导航栏拥挤

**根因**：6个标签 + 标题挤在一行

**解决方案**：
- 拆成两行：第1行标题 + 手册按钮，第2行 tab 横向滚动
- CSS `overflow-x: auto; white-space: nowrap` + 隐藏滚动条

---

### 11. 照片墙四列在手机上太密

**解决方案**：改成 `column-count: 2`，380px 以下才回退到单列

---

### 12. Render 免费版 15 分钟不访问会休眠

**解决方案**：在 [cron-job.org](https://cron-job.org) 创建每10分钟的 ping 任务
