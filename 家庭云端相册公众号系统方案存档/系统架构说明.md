# 家庭云端相册公众号系统 — 系统架构说明

## 一、系统定位

一个基于微信公众号的个人家庭照片共享系统。8个家庭成员通过公众号发送照片自动上墙，支持网页端照片墙展示、作品集分类、留言互动、按日播报。全程免费运行，无需服务器维护。

## 三、数据模型

### 照片 (Photo)
```
id: 时间戳+随机串
original_path: Cloudinary public_id
original_url:  Cloudinary 原始图片 URL
thumbnail_url: Cloudinary 缩略图 URL (w=400)
uploader_openid: 微信用户标识
uploader_nickname: 用户昵称
uploaded_at: 上传时间
month_key: 月份归档 (YYYY-MM)
category: 作品集分类 (yoyo|zhizhi|everyone|explore|'')
comments: [{author, text, created_at}]
```

### 成员 (Member)
```
openid: 微信用户标识
nickname: 昵称
added_at: 加入时间
```

### 作品集分类
| 分类 | Key | 说明 |
|---|---|---|
| 悠悠作品集 | yoyo | 悠悠的绘画手工 |
| 之之作品集 | zhizhi | 之之的创意成长 |
| 大家的作品 | everyone | 共同创作 |
| 探索发现 | explore | 动植物风景 |

## 四、核心交互流程

### 家人加入流程
```
关注公众号 → 发暗号 → 反问昵称 → 回复昵称 → 加入白名单
```

### 发照片流程
```
发图片消息 → 服务器收到XML → 验证msgId去重 → 检查白名单
→ 返回xmlReply确认语(含照片墙链接)
→ 后台异步：下载微信图片 → 压缩 → 上传Cloudinary → 写入photos.json
```

### 查看照片墙
```
访问网站 / 点公众号链接 → React加载首页 → 调用 /api/photos
→ 瀑布流展示 → 点击进入灯箱(放大+留言+保存)
```

## 五、关键技术决策与避坑

| 决策点 | 方案 | 原因 |
|---|---|---|
| 图片存储 | Cloudinary 25GB免费 | 无需信用卡，S3兼容，自带CDN |
| 数据备份 | 本地JSON + Cloudinary raw双向 | 三重试+取数据量多者恢复 |
| 微信回复 | express.text中间件 | 避免req.on('data')异步竞争 |
| 消息去重 | msgId + 5分钟Set | 微信5秒超时重试导致重复 |
| 回复确认 | xmlReply立即回复 | handleImageMessage异步处理 |
| 暗号加入 | 先匹配暗号→反问昵称→保存 | 省去管理员手动填OpenID |
| 双列布局 | CSS column-count:2 | 不对称瀑布流更自然 |
| 导航适配 | 标题+Tab横向滚动 | 手机端不拥挤 |
| 防休眠 | cron-job.org每10分钟ping | 免费版15分钟休眠 |
| 最新公众号 | 使用任一家庭成员微信注册 | 注册人不能自己发消息(微信限制) |
